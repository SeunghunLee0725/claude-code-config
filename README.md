# Claude Code Config

Claude Code (Anthropic CLI) optimized configuration based on source code analysis.

71 auto-allow rules for read-only operations, 16 deny rules for dangerous commands, global coding standards, and language-specific rules.

## What's Included

```
claude-code-config/
├── settings.json   # Permission automation + performance optimization
├── CLAUDE.md       # Global instructions (applied to all projects)
├── install.sh      # Automated installer with backup & merge
└── rules/          # Coding standards per language
    ├── common/     # Language-agnostic (always installed)
    ├── typescript/
    ├── python/
    ├── golang/
    ├── rust/
    ├── swift/
    ├── java/
    ├── kotlin/
    ├── cpp/
    ├── csharp/
    ├── perl/
    └── php/
```

## Quick Install

```bash
git clone https://github.com/SeunghunLee0725/claude-code-config.git
cd claude-code-config
chmod +x install.sh

# Install everything (all languages)
./install.sh

# Or install specific languages only
./install.sh typescript python
```

The installer will:
1. Back up your existing `~/.claude/settings.json`, `CLAUDE.md`, and `rules/`
2. Merge permission rules with your existing settings (preserving your plugins)
3. Install global instructions and coding standards

## What It Does

### settings.json

#### Auto-Allow (71 rules) - No more clicking "Allow" for:
- **Git read operations**: `git status`, `git log`, `git diff`, `git branch`, `git show`, `git blame`
- **File inspection**: `ls`, `cat`, `head`, `tail`, `wc`, `stat`, `file`, `tree`
- **System info**: `pwd`, `echo`, `date`, `env`, `which`, `du`, `df`
- **Package managers**: `npm list/info/view`, `pip list/show`, `go list`, `cargo --version`
- **GitHub CLI**: `gh pr list/view/status/checks`, `gh issue list/view`, `gh repo view`
- **Text processing**: `jq`, `sort`, `uniq`, `cut`, `awk`, `sed -n`, `diff`
- **Built-in tools**: `Read`, `Glob`, `Grep`, `WebFetch`, `WebSearch`

#### Auto-Deny (16 rules) - Blocked even in bypass mode:
- `rm -rf /`, `rm -rf ~`, `rm -rf /*`
- `git push --force origin main/master`
- `git reset --hard origin/main/master`
- `git clean -fdx`
- `chmod -R 777`, fork bomb, `mkfs`, `dd` to device
- `curl|bash`, `wget|bash` pipe execution
- `pip install --break-system-packages`

#### Performance Settings
- `autoMemoryEnabled`: Auto-save learned patterns across sessions
- `alwaysThinkingEnabled`: Deep reasoning on every query
- `attribution: ""`: Clean git history (no "Generated by Claude" footers)
- `worktree.symlinkDirectories`: Avoid duplicating `node_modules`, `__pycache__`, `.venv`, etc.

### CLAUDE.md - Global Instructions

Applied to every session regardless of working directory:
- Concise response style (no preamble, no trailing summaries)
- Immutability-first coding, small files/functions
- TDD workflow (write tests first, 80%+ coverage)
- Conventional commits, no force push to main
- Security-first (no hardcoded secrets, parameterized queries)
- Proactive `/compact` for context window management

## Configuration Priority

Claude Code loads configs in this order (later = higher priority):

```
/etc/claude-code/CLAUDE.md          (system, lowest)
~/.claude/settings.json             (user global) ← installed here
~/.claude/CLAUDE.md                 (user global) ← installed here
~/.claude/rules/*.md                (user rules)  ← installed here
.claude/settings.json               (project)
CLAUDE.md                           (project)
.claude/rules/*.md                  (project rules)
CLAUDE.local.md                     (local, highest)
```

Project-level files override user-level when both exist.

## Optional: Language Setting

Add to your `~/.claude/settings.json` to get responses in your preferred language:

```json
{
  "language": "korean"
}
```

Available: `korean`, `japanese`, `chinese`, `spanish`, `french`, `german`, etc.

## Customization

### Add project-specific overrides

Create `CLAUDE.local.md` in your project root (gitignored):

```markdown
# Project-specific rules
- Use pnpm instead of npm
- Database: PostgreSQL with Prisma ORM
- Deployment: AWS ECS with Terraform
```

### Add conditional rules

Create `.claude/rules/react.md` in your project:

```yaml
---
paths: "src/components/**/*.tsx, src/hooks/**/*.ts"
description: "React component rules"
---
- Use functional components with hooks
- Colocate tests with components
- Use React.memo for expensive renders
```

## Uninstall

```bash
# Restore from backup
ls ~/.claude/backups/
cp ~/.claude/backups/<timestamp>/settings.json ~/.claude/settings.json
cp ~/.claude/backups/<timestamp>/CLAUDE.md ~/.claude/CLAUDE.md
cp -r ~/.claude/backups/<timestamp>/rules ~/.claude/rules
```
